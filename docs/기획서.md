# 🎮 steam-reviewer 기획서

> 게임 이름만 넣으면, 리뷰 수천 개를 분석해 "플레이어가 뭘 좋아하고 싫어하는지"를
> 통계 + AI 요약 리포트로 뽑아주는 도구

---

## 1. 한 줄 정의

> **게임 이름(또는 Steam App ID)을 넣으면, Steam 리뷰를 대량 수집·분석해서 통계 진단 + AI 자연어 요약 리포트(텍스트/HTML)를 자동 생성하는 Python 라이브러리/CLI**
>
> 데이터 소스: Steam 공개 리뷰 API (키 불필요, read-only)

---

## 2. 왜 만드는가 (동기)

- **개인 목표**: `pip install` 한 줄로 가져다 쓰는 오픈소스. 대량 텍스트 NLP + LLM 요약을 제대로 공부.
- **시장 빈틈**:
  - Steam 리뷰는 수천~수만 개라 사람이 다 읽을 수 없음. 별점/추천%만으론 "왜"를 모름.
  - "이 게임의 칭찬·불만 포인트를 측면별로 요약"해주는 가벼운 오픈소스 도구는 드뭄 → 차별점.
- **수요**: 게이머·개발자·리뷰어 모두 "이 게임 평가 핵심만 빠르게" 원함. 영어권이라 도달 범위 넓음.

---

## 3. 타겟 사용자

1. 살까 말까 고민하는 **게이머** (핵심만 빠르게)
2. 자기 게임 피드백을 분석하려는 **인디 게임 개발자**
3. 대량 텍스트 분석을 학습하려는 **데이터 분석/AI 입문자**

---

## 4. 핵심 기능

### 입력 (Loaders)
- `resolve_appid(name)` — 게임 이름 → App ID (Steam store search, 키 불필요)
- `fetch_reviews(appid, ...)` — 커서 페이지네이션으로 리뷰 대량 수집
- `fetch_appdetails(appid)` — 게임명·장르·헤더 이미지·가격

### 분석 (Analyzers) — *AI 없이 순수 분석으로 동작*
- **기본 통계**: 긍/부정 비율, 리뷰 수, 평균 플레이타임, 추천자 vs 비추천자 플레이타임 비교
- **키워드 추출**: 긍정/부정 리뷰의 빈출 키워드 (불용어 제거)
- **시간 추세**: 작성일별 리뷰 수·감성 변화 (패치 전후 변화 포착)
- **도움됨(votes_up) 가중**: 영향력 큰 리뷰 식별

### AI 요약 (AI Layer) — *분석 위에 얹는 선택 계층*
- LLM(Gemini 무료 티어)이 리뷰를 **측면별(게임성/성능/스토리/가격/조작)**로 칭찬·불만 요약
- 키 없으면 통계만 출력 (선택 기능)

### 출력 (Report)
- 텍스트 리포트 (기본) + HTML 리포트 (헤더 이미지·차트 포함)

---

## 5. 범위 (Scope)

### ✅ In scope
- Steam 공개 리뷰 수집·분석
- 감성·키워드·시간추세 분석
- AI 측면별 요약
- 텍스트/HTML 리포트 + 결과 캐싱

### ❌ Out of scope
- 리뷰 작성/투표 등 쓰기 동작 (read-only)
- 리뷰 원문·작성자 정보 재배포 (집계·요약만)
- "이 게임 사라/사지마라" 식 추천 (→ "플레이어 평가 진단"까지만)
- 웹/앱 UI (라이브러리 + CLI 유지)

---

## 6. 핵심 안전·매너 원칙

1. **레이트리밋 준수 + 캐싱** — Steam 서버에 무리 주지 않음 (요청 간 간격, 로컬 캐시)
2. 리뷰는 공개 데이터지만 **작성자 개인정보(steamid 등)는 저장·재배포 안 함** — 집계·요약만
3. **"추천" 아니라 "진단·요약"** — "플레이어들이 이렇게 평가한다"까지만
4. `analyzers`는 AI 없이도 동작 — AI는 선택 계층
5. AI 키(Gemini)는 `.env`에만, 깃 커밋 금지

---

## 7. 기술 스택

| 영역 | 선택 |
|---|---|
| 언어 | Python 3.10+ |
| HTTP | requests / httpx |
| 분석 | pandas, 표준 라이브러리(collections 등) |
| 키워드 | 간단 빈도 기반 → (추후 형태소/TF-IDF) |
| 차트 | matplotlib (선택 의존성) |
| AI | Gemini API (`google-genai`, 무료 티어) — 선택 계층 |
| CLI | typer |
| 캐싱 | 로컬 JSON 캐시 |
| 테스트 | pytest |
| 배포 | pyproject.toml + PyPI |

## 8. 폴더 구조

```
steam-reviewer/
├── steam_reviewer/
│   ├── __init__.py
│   ├── loaders/
│   │   └── steam.py        # 리뷰/앱정보 수집, App ID 해석
│   ├── analyzers/
│   │   ├── basic.py        # 긍부정/플레이타임 통계
│   │   ├── keywords.py     # 키워드 추출
│   │   └── trends.py       # 시간대별 추세
│   ├── ai/
│   │   └── summarize.py    # LLM 측면별 요약
│   ├── report/
│   │   ├── text.py
│   │   └── html.py
│   ├── cache.py            # 로컬 캐시
│   └── cli.py
├── tests/
├── examples/
├── docs/
│   └── 기획서.md
├── .env.example
├── .gitignore
├── README.md
└── pyproject.toml
```

> 설계 원칙: **`loaders`/`analyzers`는 AI 없이도 완전히 동작**. AI는 위에 얹는 선택 계층.
